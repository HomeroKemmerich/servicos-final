Web Services

Principais diferenças entre os frameworks Arduino e ESP-IDF para ESP32 na utilização de serviços web

Diferenças gerais

A primeira diferença pode ser encontrada na linguagem. Enquanto o framework Arduino utiliza a linguagem C++, o framework ESP-IDF implementa diretamente a linguagem C. De forma geral a linguagem C tende a ter um melhor desempenho e maior controle sobre as tarefas executadas pelo microcontrolador, porém isso vem a um preço de exigir mais configurações manuais.

Funções de controle

No ESP-IDF, o void app_main() é a função principal do programa. É onde o código principal do aplicativo é escrito. É semelhante ao void setup() e void loop() do Arduino, mas existem algumas diferenças importantes.

  1. Estrutura do programa: No ESP-IDF, o programa é estruturado de forma mais modular, com várias funções sendo definidas e chamadas a partir da função app_main(). Isso permite uma organização mais flexível do código e facilita a reutilização de partes específicas do programa. No Arduino, o programa é estruturado em duas funções principais: app_main() e app_main(), onde o código de inicialização é colocado no app_main() e o código principal é colocado no app_main().
  2. Execução do programa: No ESP-IDF, o app_main() é executado apenas uma vez, quando o programa é iniciado. Após a execução do app_main(), o programa é encerrado. No Arduino, o app_main() é executado uma vez, no início do programa, e o app_main() é executado continuamente em um loop infinito.
  3. Manipulação de eventos: No ESP-IDF, é comum usar a biblioteca de eventos do FreeRTOS para lidar com eventos assíncronos, como interrupções, temporizadores e comunicação com periféricos. Esses eventos são tratados em funções separadas, que são chamadas a partir do app_main(). No Arduino, a manipulação de eventos é geralmente feita dentro do app_main(), onde o código é executado repetidamente.
  4. Acesso a recursos do sistema: No ESP-IDF, é possível acessar recursos do sistema, como GPIOs, UARTs, SPIs, etc., diretamente no app_main() ou em funções chamadas a partir dele. No Arduino, o acesso a esses recursos é feito principalmente dentro do app_main() e app_main(), usando funções e bibliotecas específicas do Arduino.

Em resumo, enquanto o void app_main() do ESP-IDF é a função principal do programa, onde o código principal é escrito e chamadas para outras funções são feitas, o void setup() e void loop() do Arduino são usados para inicialização e execução contínua do código principal, respectivamente. O ESP-IDF oferece uma estrutura mais modular e flexível, enquanto o Arduino é mais simples e voltado para projetos de prototipagem rápida.

Conexão WiFi

O framework Arduino possui uma biblioteca própria para a utilização de serviços web, enquanto o framework ESP-IDF utiliza a biblioteca esp_wifi.

main.c


                #include 
                #include "lwip/apps/sntp.h"
    
                void wifi_connection()
                {
                    // 1 - Wi-Fi/LwIP Init Phase
                    esp_netif_init();                    // TCP/IP initiation 					s1.1
                    esp_event_loop_create_default();     // event loop 			                s1.2
                    esp_netif_create_default_wifi_sta(); // WiFi station 	                    s1.3
                    wifi_init_config_t wifi_initiation = WIFI_INIT_CONFIG_DEFAULT();
                    esp_wifi_init(&wifi_initiation); // 					                    s1.4
                    // 2 - Wi-Fi Configuration Phase
                    esp_event_handler_register(WIFI_EVENT, ESP_EVENT_ANY_ID, wifi_event_handler, NULL);
                    esp_event_handler_register(IP_EVENT, IP_EVENT_STA_GOT_IP, wifi_event_handler, NULL);
                    wifi_config_t wifi_configuration = {
                        .sta = {
                            .ssid = WIFI_SSID,
                            .password = WIFI_PASS
                        }
                    };
                    esp_wifi_set_config(ESP_IF_WIFI_STA, &wifi_configuration);
                    // 3 - Wi-Fi Start Phase
                    esp_wifi_start();
                    // 4- Wi-Fi Connect Phase
                    esp_wifi_connect();
                }
            
                void app_main(void){
                    wifi_connection();
                }

Conexão HTTP

Para a conexão HTTP, o framework Arduino utiliza a biblioteca HTTPClient, enquanto o framework ESP-IDF utiliza a biblioteca esp_http_client.

main.c


                #include "esp_http_client.h"

                static void rest_get(char *path)
                {
                    char url[100];
                    sprintf(url, "%s%s", BASE_URL, path);
                    strcpy(url, BASE_URL);
                    strcat(url, path);
                
                    esp_http_client_config_t config_get = {
                        .url = url,
                        .method = HTTP_METHOD_GET,
                        .cert_pem = NULL,
                        .event_handler = client_event_get_handler};
                    
                    esp_http_client_handle_t client = esp_http_client_init(&config_get);
                    esp_http_client_perform(client);
                    esp_http_client_cleanup(client);
                }

Conclusão

Apesar de ambos os frameworks possuírem bibliotecas para a utilização de serviços web, o framework ESP-IDF possui uma maior flexibilidade e controle sobre o código, enquanto o framework Arduino possui uma maior facilidade de utilização.
